Payment Receipt - Flow

Generally, payment receipt should be created after we receive callback from payment gateway (e.g. Xendit). However, there are certain cases where we might not need to use Xendit, the clear example would be if a user can purchase subscription for free; whilst free, payment receipt should still be created as transaction proof. Hence, there are two main entry point to create payment receipt.

  1. Via webhook (main)
  2. Directly after purchase request is created
    • main example: free subscription
